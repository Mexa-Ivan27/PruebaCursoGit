COMANDOS:
git clean: elimina solo los archivos no trackeados del working directory
git clean -f: forza a eliminar el archivo no trackeado
git clean -n: Muestra todo lo que se va a borrar
git restore <archivo>: Este comando permite reestablecer un cambio hecho en el codigo

git restore --staged <file>: sirve para bajar "algo" de el staging (etapa 2) al working directory (etapa 1)



CREANDO RAMAS
git branch: Con este comando podemos ver cuantas ramas existen y en cuál estamos parados
git branch <name>: Con este comando creamos una rama
git branch -d <name>: Eliminas una rama
git branch -m <old_name> <new_name>: Cambias el nombre
git checkout <commit_id>: Navegas al cambio que hiciste en ese commit
git checkout <name_branch>: Te permite cambiar de una rama a otra 
git checkout -b <banch_name>: Crea una rama y te cambia a esa rama 


GIT MERGE
git merge <branch_name>: Tienes que estar en la rama master / main para unir el código (Se une con todo y commits)


CONFLICTOS
Cuando 2 desarrolladores escriben en la misma rama
1 se puede dar cuando inicias un proceso de merge
y otro es durante el proceso de merge 


STASH: sirve para guardar una porción de código (funciona como una pila en programación)
git stash: guardas los últimos cambios
git stach pop: traes los cambios guardados
git stash list: Muestra todos los git stash guardados


TAG: sirve para etiquetar un commit como importantes (Cuando creas un tag apunta a un solo commit)
git tag <comentario>: tag ligero
git tag -a v1.0: tag anotado
git tag: enlista todos los tag
git tag -a version0.0 -m <comentario> <hash>
git checkout <tag_name>: viaja en el tiempo al checkout
git show <tag_name>: Muestra quien lo hizo
git tag -d <tag_name>: Elimina un tag
